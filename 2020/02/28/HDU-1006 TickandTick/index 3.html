HDU 1006 TickandTick

Feb 28, 2020

Time Limit: 2000/1000 MS (Java/Others) Memory Limit: 65536/32768 K (Java/Others)
Total Submission(s): 26419 Accepted Submission(s): 7236

Problem Description

The three hands of the clock are rotating every second and meeting each other many times everyday. Finally, they get bored of this and each of them would like to stay away from the other two. A hand is happy if it is at least D degrees from any of the rest. You are to calculate how much time in a day that all the hands are happy.

Input

The input contains many test cases. Each of them has a single line with a real number D between 0 and 120, inclusively. The input is terminated with a D of -1.

Input

For each D, print in a single line the percentage of time in a day that all of the hands are happy, accurate up to 3 decimal places.

Sample Input

0
120
90
-1

Sample Output

100.000
0.000
6.251

解析

首先0~12和12～24完全相似，缩短一半时间
然后求出时分秒指针各自的角速度（rad/s）

hourAngularVelocity = 360.0 / (12 * 60 * 60)
minuteAngularVelocity = 360.0 / (60 * 60)
secondAngularVelocity = 360.0 / 60

随后求出每两指针的相对角速度

relativehm = mAngularVelocity - hAngularVelocity
relativehs = sAngularVelocity - hAngularVelocity
relativems = sAngularVelocity - mAngularVelocity

则每两指针的要求是（两指针角度>=D 并且 (360-两指针角度>D)）
即 D <= 两指针角度差 <= 360-D
由于角度有周期性
则 360 *（i-1）+ D <= 两指针角度差 <= 360 * i + D
由于两角度指针差 = relativehm * 时间 （时间的范围是我们需要求的，所以设为l~r）
则

360 * (i-1) + D<= relativehm * (l1 ~ r1) <= 360 * i-D
360 * (j-1) + D<= relativehs * (l2 ~ r2) <= 360 * j-D
360 * (k-1) + D<= relativems * (l3 ~ r3) <= 360 * k-D

此处l1,l2,l3,r1,r2,r3 均在[0 ~ 12 * 60 * 60]
所以问题转换成了求每一个i,j,k时l1 ~ r1, l2 ~ r2, l3 ~ r3的交集的问题

我的错误

  4. 注意该代码求出的r们需要判断是否大于上界12 * 60 * 60，大于则取等号
